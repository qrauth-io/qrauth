Trust Reveal UX ​

When a user scans a QRAuth-signed QR code, they land on the verification page at https://qrauth.io/v/:token. The page does not simply display information — it stages a 3-second animated reveal that communicates trust level before showing any details.

The reveal is implemented entirely in CSS animations and vanilla JS. There is no external animation library.

Success flow ​

1. Shield pop-in      (~400ms)
2. Trust color sweep  (~800ms)   green / amber / red
3. Organization name  (~300ms)
4. Visual fingerprint (~500ms)
5. Fade to detail view

Step-by-step ​

1. Shield pop-in A shield icon scales in from 0 to 1 with a cubic-bezier(0.175, 0.885, 0.32, 1.275) spring. The shield color is neutral (white/dark) until the trust color is determined.

2. Trust color sweep The background performs a radial sweep from the center of the shield outward. The sweep color maps to the trust score:

Trust scoreColorMeaning
80–100#00a76f (green)Trusted
50–79#ffab00 (amber)Caution
< 50#ff5630 (red)Suspicious / fraudulent

3. Organization name The verified organization name fades in below the shield, accompanied by its trust level badge (GOVERNMENT, BUSINESS, INDIVIDUAL) and KYC status.

4. Visual fingerprint A deterministic SVG pattern generated from the scan's HMAC seed appears as a 6×6 grid of colored cells. The same scan always produces the same pattern; a cloned page cannot reproduce it.

5. Detail view The overlay fades out and the full verification detail appears: destination URL, location match, content type, and fraud signal summary.

Failure flow ​

When trustScore < 50 or the signature is invalid:

  1. Red flash — full-screen crimson background pulse (2 cycles, 300ms each).
  2. Alarm text — FRAUDULENT CODE DETECTED rendered in oversized bold, centered.
  3. Pulse animation — the warning text pulses at 1.5Hz until dismissed.
  4. No destination URL is shown — the page never reveals where the code would have taken the user.
css
@keyframes alarm-pulse {
  0%, 100% { opacity: 1; transform: scale(1); }
  50%       { opacity: 0.7; transform: scale(1.04); }
}

.alarm-text {
  animation: alarm-pulse 666ms ease-in-out infinite;
  color: #fff;
  font-size: clamp(1.8rem, 6vw, 3rem);
  font-weight: 900;
  text-align: center;
}

Visual fingerprint ​

The fingerprint SVG is deterministic: it is generated by seeding a simple PRNG with the scan's per-scan HMAC value. Every scan of the same code at the same time produces the same fingerprint. A phishing page cloning the visual cannot reproduce it because it does not have access to the HMAC.

ts
function generateFingerprint(hmacHex: string): string {
  // Seed LCG with first 8 bytes of HMAC
  let seed = parseInt(hmacHex.slice(0, 8), 16);

  const cells: string[] = [];
  const palette = ['#00a76f', '#0066cc', '#ff6b35', '#9b59b6', '#1abc9c', '#e74c3c'];

  for (let i = 0; i < 36; i++) {
    // LCG step
    seed = (seed * 1664525 + 1013904223) & 0xffffffff;
    const colorIdx = Math.abs(seed) % palette.length;
    const opacity = 0.3 + (Math.abs(seed >> 8) % 70) / 100;
    cells.push(`<rect x="${(i % 6) * 16}" y="${Math.floor(i / 6) * 16}"
      width="15" height="15" rx="2"
      fill="${palette[colorIdx]}" opacity="${opacity}"/>`);
  }

  return `<svg viewBox="0 0 96 96" xmlns="http://www.w3.org/2000/svg">${cells.join('')}</svg>`;
}

Ephemeral proof ​

Every verification page shows an ephemeral proof block: a set of facts that are unique to this specific scan and cannot be reproduced by a cloned page.

FieldValueWhy it proves authenticity
Citye.g. "Athens, GR"Resolved from scan IP; cloner cannot fake it
Devicee.g. "iPhone 15 · Safari 17"Parsed from User-Agent at scan time
Timestampe.g. "10 Apr 2026, 14:32:07"Server-recorded; not client-supplied
HMAC fingerprint16-char hexUnique per scan; not in the QR payload

Origin integrity check

The verification page JavaScript validates window.location.hostname against the expected domain. If it does not match qrauth.io, the page renders a phishing warning instead of the normal reveal, even if all visual assets were copied.

Trust score coloring reference ​

ts
function trustColor(score: number): string {
  if (score >= 80) return '#00a76f'; // green
  if (score >= 50) return '#ffab00'; // amber
  return '#ff5630';                  // red
}

The same color is used for:

  • The shield fill during the color sweep.
  • The trust badge background in the detail view.
  • The border on the visual fingerprint frame.

Mobile design ​

The reveal is designed for portrait mobile viewports (360px+):

  • All animations use transform and opacity only (no layout-triggering properties).
  • Font sizes use clamp() to scale between 360px and 768px.
  • The shield SVG is 100px × 100px on mobile, 140px × 140px on desktop.
  • Touch-to-dismiss collapses the reveal on small screens.
